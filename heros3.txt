1. Настройка LAPS
Local Administrator Password Solution (LAPS) позволяет централизованно управлять паролями локальных администраторов на компьютерах в домене.

Установка LAPS на контроллер домена (WIN-RIGHT-1):

Скачайте и установите LAPS на контроллер домена.
Установите компоненты LAPS: Group Policy Management Console, Fat32 format utility, PowerShell module, and Group Policy Client-Side Extension (CSE).
Настройка групповой политики для LAPS:

Откройте "Group Policy Management" и создайте новую политику или отредактируйте существующую.
Перейдите в "Computer Configuration" -> "Policies" -> "Administrative Templates" -> "LAPS".
Настройте параметры:
Enable local admin password management: Enabled.
Password Settings: установите длину пароля, сложность и срок действия.
Name of Administrator account to manage: Administrator.
Примените политику к нужным OU.
Проверка работы LAPS:

Перезагрузите клиентский компьютер (CLI-WIN).
Убедитесь, что пароль локального администратора изменился и отображается в Active Directory.
2. Настройка стартовой страницы и закладок в браузере Edge
Настройка стартовой страницы:

Откройте "Group Policy Management" и создайте новую политику или отредактируйте существующую.
Перейдите в "User Configuration" -> "Policies" -> "Administrative Templates" -> "Windows Components" -> "Microsoft Edge".
Настройте параметр "Configure the enterprise mode site list": Enabled.
Укажите стартовую страницу: https://rds.rea2025.skills.
Добавление закладок:

В той же политике настройте параметр "Configure the Enterprise Mode Site List": Enabled.
Добавьте закладку: https://web-pecypca.

Откройте Group Policy Management Console (GPMC):

На контроллере домена (WIN-RIGHT-1) откройте "Group Policy Management Console".
Создайте или отредактируйте групповую политику:

Создайте новую групповую политику или отредактируйте существующую, которая будет применяться к нужным организационным единицам (OU), содержащим компьютеры пользователей.
Настройка стартовой страницы:

Перейдите в "User Configuration" -> "Policies" -> "Administrative Templates" -> "Windows Components" -> "Microsoft Edge".
Найдите параметр "Configure the enterprise mode site list" и включите его.
Укажите стартовую страницу: https://rds.rea2025.skills.
Добавление закладок:

В той же групповой политике найдите параметр "Configure the Enterprise Mode Site List" и включите его.
Добавьте закладку: https://web-pecypca.
Примените групповую политику:

Примените созданную или отредактированную групповую политику к нужным OU.
Проверка настроек:

Перезагрузите клиентский компьютер (CLI-WIN) или обновите групповую политику на нем с помощью команды gpupdate /force.
Убедитесь, что стартовая страница и закладки настроены корректно в браузере Edge.

3. Создание папки с ограниченным доступом
Создание папки:

На контроллере домена (WIN-RIGHT-1) создайте папку C:\DACL.
Настройка прав доступа:

Откройте свойства папки и перейдите на вкладку "Security".
Настройте права доступа:
Администраторы: Full Control.
Пользователи домена с полем title "portal": Modify.
4. Настройка VPN Always On
Установка и настройка RRAS (Routing and Remote Access Service):

На сервере WIN-MID установите роль "Remote Access".
Настройте VPN-сервер:
Создайте VPN-подключение с использованием IKEv2.
Настройте сертификаты для аутентификации.
Настройка автоматического подключения VPN:

Настройте групповую политику для автоматического подключения VPN.
Перейдите в "Computer Configuration" -> "Policies" -> "Windows Settings" -> "Security Settings" -> "Network List Manager Policies".
Создайте новую политику и настройте автоматическое подключение VPN.
Настройка публикации списков отзыва сертификатов (CRL):

Настройте публикацию CRL на сервере WIN-MID.
Убедитесь, что CRL доступны из интернета.
Ограничение доступа через VPN:

Настройте маршрутизацию и брандмауэр на сервере WIN-RTR для ограничения доступа через VPN только к контроллеру домена (DC).
5. Настройка RDS (Remote Desktop Services)
Настройка сертификатов для RDS:

На сервере WIN-RIGHT-2 настройте использование корпоративного центра сертификации для обеспечения безопасного подключения по протоколу HTTPS.
Настройка SSO (Single Sign-On):

Настройте SSO для RDS, чтобы пользователи могли подключаться без ввода логина и пароля.
Настройка Web Application Proxy:

На сервере WIN-MID-2 установите и настройте роль Web Application Proxy.
Настройте публикацию RDS через Web Application Proxy.
Распространение ярлыка на опубликованное приложение:

Настройте групповую политику для распространения ярлыка на опубликованное приложение (HMM3 HOTA) для пользователей группы безопасности REA2025\gamers.
6. Проверка и тестирование
Проверка всех настроек:

Убедитесь, что все сервисы и устройства работают корректно после перезагрузки.
Проверьте доступность всех ресурсов через VPN и RDS.
Документирование:

Оставьте черновик с использованными учетными данными, адресами и DNS именами на рабочем месте.
Документируйте все выполненные шаги и настройки.
